# تست‌های خودکار ربات رهیار قانون

## این پوشه چیه؟

هر بار که یک قابلیت جدید به ربات اضافه می‌شه یا یک باگ رفع می‌شه، این‌جا یک
تست کوچیک نوشته می‌شه که رفتار درست اون بخش رو تضمین می‌کنه. تست‌ها ربات رو
واقعاً به بله وصل نمی‌کنن — به‌جاش وانمود می‌کنن یک کاربر داره پیام می‌فرسته
و چک می‌کنن نتیجه (پیام‌های ارسالی، تغییرات دیتابیس) درست باشه یا نه.

## چرا مهمه؟

فرض کنید ماه دیگه بخواید یک تغییر کوچیک بدید (مثلاً متن یک پیام رو عوض کنید)
ولی این تغییر تصادفاً یک قابلیت دیگه (مثلاً اعمال کد تخفیف) رو خراب کنه. بدون
تست، این خرابی رو فقط وقتی می‌فهمید که یک مشتری واقعی شکایت کنه. با تست‌های
خودکار، در چند ثانیه و **قبل از آپلود روی Railway** متوجه می‌شید.

## چطور اجراشون کنم؟

روی کامپیوتر خودتون (یا از طریق Codespaces/Replit اگه به پایتون دسترسی
ندارید):

```bash
pip install pytest
pytest tests/
```

خروجی چیزی شبیه این می‌ده:

```
tests/test_bot.py::test_registration_completes PASSED
tests/test_bot.py::test_invalid_national_id_is_rejected PASSED
tests/test_bot.py::test_question_flow_with_review_step PASSED
...
10 passed in 0.42s
```

اگه یک خط قرمز (`FAILED`) دیدید، دقیقاً می‌گه کدوم تست و کدوم خط از کد مشکل
داره — همون‌جا شروع کنید به بررسی.

## چه زمانی باید تست جدید اضافه کنم؟

هر وقت یک قابلیت مهم جدید اضافه کردید (یا یک باگ جدی رفع کردید)، خوبه یک
تابع کوچیک `test_...` به `test_bot.py` اضافه کنید که همون رفتار رو چک کنه.
لازم نیست پیچیده باشه — همین که چک کنه «بعد از این عملیات، وضعیت درست شد یا
نه» کافیه.
